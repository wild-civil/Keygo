# KeyGo · BLE 智能车钥匙系统

> 一套开源的 **BLE 舒适进入（Passive Keyless Entry）** 解决方案：手机靠近车辆自动解锁、离开自动锁车，**零后台额外功耗**，并在荣耀 / 小米等深度管控机型上最大化进程存活率。

KeyGo 由两部分组成：

- **固件**：基于 **WCH CH582M** 蓝牙 SoC，接管原车钥匙的解锁 / 锁车 / 后备箱三个物理按键
- **App**：基于 **uni-app（Vue 3）**，通过 BLE 与固件通信，并实现「智能重连」引擎

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🔑 **舒适进入** | 走近自动解锁、走远自动锁车，无需掏出手机 |
| 🔄 **智能重连（三种模式）** | 极速 / 舒适 / 省电，用户自选，覆盖不同续航与体验诉求 |
| 📶 **RSSI 实时信号** | 通过 BLE Notify 实时接收设备信号强度，辅助距离判断 |
| 🔓 **手动控制** | App 一键解锁 / 锁车 / 开启后备箱 |
| ⚙️ **参数配置** | RSSI 阈值、确认次数、采样间隔等可调并下发到设备 |
| 🔋 **电池服务** | 固件上报电压 / 电量，App 展示并低电提醒（v3.14+） |
| 🛡️ **设备绑定** | MAC 白名单 + 设备唯一名，绑定后仅授权手机可控制 |
| 🚀 **前台保活** | 原生前台服务 + 电池优化豁免引导，对抗系统后台查杀 |
| 🧭 **GPS 围栏（极速模式）** | 停车位置自动建围栏，走到车边即连好 |

## 🔄 智能重连：三种模式（v3.23 / v1.0.1）

这是本项目最关键的工程创新。不同用户对「自动重连」的诉求完全不同，因此提供三档模式让用户自选。

| | 极速模式 | 舒适模式（默认） | 省电模式 |
|:--|:--|:--|:--|
| **恢复延迟** | 走到车边已就绪 | 亮屏后 2–8 秒 | 需打开 App |
| **额外功耗** | ≈ 0.45%/天 | **≈ 0.08%/天** | 0 |
| **额外权限** | 后台定位 | 无 | 无 |
| **检测原理** | 加速度计 + GPS 围栏 + 心跳兜底 | 亮屏/解锁即触发扫描 | onShow 扫描 |
| **后台轮询** | 无 | 无（完全事件驱动） | 无 |
| **适合人群** | 每日通勤、追求无感 | 大部分用户 ★默认 | 低频 / 电量焦虑 |

### 舒适模式 · 零后台功耗（v1.0.1 重构重点）

**设计理念**：不做任何后台轮询，完全事件驱动。旧方案「每 2 分钟定时扫描」在荣耀等深度管控机型上，锁屏 2h+ 后因「高耗电应用」判定被杀，重连彻底哑火。

```
用户从口袋掏出手机
  → 抬手亮屏 / 按电源键 / 指纹解锁
  → BroadcastReceiver: ACTION_SCREEN_ON / ACTION_USER_PRESENT
  → JS 回调 _onScreenOn()（2s 防抖，合并重复事件）
  → 前置检查：已连接？蓝牙关？30s 内扫过？有目标设备？
  → 启动 8s BLE 扫描（service UUID 过滤）
  → 发现目标 → 连接成功才停扫描（不提前停，留重试余地）
  → 8s 内未连上 → 30s 后二阶段重试，最多 3 次
```

- **零后台额外功耗**：不持有 WakeLock、无 setInterval，系统管家无可标记项
- **进程存活最大化**：仅亮屏瞬间的动态 Receiver 触发，系统找不到杀你的理由
- **连接可靠**：扫描到 ≠ 连上，必须等连接确认才停扫描；失败可原地重连

### 极速模式 · 运动感知 + GPS 围栏 + 心跳兜底

三层触发器「或」逻辑，任意一层命中即启动 BLE 扫描，专为每日通勤无感体验设计：

1. **加速度计（SoC Sensor Hub，~0.5mA）**：检测到走路/拿起 → 读缓存 GPS → 在停车围栏附近才触发精确定位
2. **GPS 围栏（Google Geofence）**：进入停车围栏半径即触发
3. **AlarmManager 5min 心跳兜底**：华为/荣耀等无 GMS 设备自动降级到舒适模式

### 省电模式

仅在 App 切回前台（`onShow`）时扫描连接，零额外耗电、零权限，适合低频使用。

## 📡 BLE 通信协议

| UUID | 类型 | 方向 | 说明 |
|------|------|------|------|
| `FF00` | Service | — | KeyGo 主服务 |
| `FF01` | Write | App→设备 | 配置下发（格式：`unlock=-45 lock=-65 uc=5 lc=10 interval=500`） |
| `FF02` | Read/Notify | 设备→App | 状态上报（JSON，含 RSSI / 电量等） |
| `FF03` | Write | App→设备 | 手动命令（`UNLOCK` / `LOCK` / `TRUNK` / `STATUS` / `UNBIND`） |

- 设备名：`KeyGo-<MAC后6位>`，每个设备唯一
- 广播间隔：50ms（≈ 20 次/秒），任意 ≥1s 扫描窗口必能捕获
- 安全：MAC 白名单（NVS 持久化）+ 物理按键配对 + 恢复出厂

## 🚀 开发指南

### 固件（CH582M）

1. 安装 **MounRiver Studio**（WCH 官方 IDE）
2. 打开 `code/CH582M/CH582M_BLE_Slave/CH582M_BLE_Slave.wvproj`
3. 编译 → 下载到 CH582M 开发板（通过 WCH-Link）
4. 设备上电后开始 BLE 广播，设备名 `KeyGo-XXXXXX`

### App（uni-app）

1. 用 **HBuilderX** 打开 `app/BLE_Key_Go_App`
2. 运行到 Android 手机（需开启开发者模式 + USB 调试）
3. 确保手机蓝牙已开启、已授予定位权限
4. 打开 App → 自动扫描 KeyGo 设备 → 点击连接 → 控制 / 配置

所需权限（`manifest.json` 已声明）：蓝牙、定位（含后台定位）、前台服务、前台服务类型 `connectedDevice`、通知、电池优化豁免请求。
