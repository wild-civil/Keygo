## 1. 项目愿景

### 核心功能

BLE-Key-Go 是一个 ESP32-C3 驱动的蓝牙车钥匙模块。它安装在车内，接管原车钥匙的（解锁/锁车/后备箱）三个物理按键，通过 BLE 与手机通信，实现：

- **自动感应**：走近自动解锁、走远自动锁车，无需掏出手机
- **手动控制**：App 一键解锁/锁车/开启后备箱
- **独占绑定**：首次扫描/扫码配对后，仅绑定的手机可连接控制
- **钥匙分享**：可生成临时授权码分享给家人朋友（规划中）

### 对标产品

"红树河熊" 等商业化 BLE 车钥匙产品——以它们的 UX 为目标，同时完全开源可控。

## 4. RSSI 管线设计

### 数据流

```
蓝牙连接信号强度
      │
      ▼
┌─────────────┐
│ raw RSSI    │  手机与 ESP32 之间的 BLE 连接 RSSI (dBm)
│ (每500ms)   │  双栈读取: Bluedroid (esp_ble_gap_read_rssi) / NimBLE (ble_gap_conn_rssi)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  尖峰丢弃    │  单次离群点（身体遮挡）→ 丢弃，不喂入 Kalman
│ 连续计数     │  N 次连续离群（真实趋势）→ 放行，喂入 Kalman
│  阈值25dB   │  可配置: cfg spike=25
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 1D Kalman   │  一维随机游走模型
│  滤波       │  Predict: p = p + q
│ q=4 r=16    │  Update:  k = p / (p + r)
│             │           x = x + k*(z - x)
└──────┬──────┘          p = (1-k) * p
       │
       ▼
┌─────────────┐
│ 滞后状态机   │  Locked → Unlock: filtered >= unlock阈值
│ dead zone   │  Unlocked → Lock: filtered <= (lock阈值 - hyst)
│ hyst=5dB    │  滞后带防止边界震荡
│             │  需连续 N 次满足才触发: uc=3, lc=5
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ GPIO 输出    │  解锁/锁车/后备箱 继电器脉冲序列
└─────────────┘
```

### 可调参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `unlock` | -45 dBm | 解锁阈值（信号 > 此值） |
| `lock` | -65 dBm | 锁车阈值（信号 < 此值 - hyst） |
| `hyst` | 5 dB | 滞后宽度（防抖） |
| `spike` | 25 dB | 尖峰丢弃容差 |
| `q` | 4.0 | Kalman 过程噪声（越大跟踪越快） |
| `r` | 16.0 | Kalman 测量噪声（越大越平滑） |
| `uc` | 3 | 解锁需连续满足次数 |
| `lc` | 5 | 锁车需连续满足次数 |
| `interval` | 500 ms | RSSI 采样间隔 |
| `dlock` | 5000 ms | 断连锁车延时 |

## 5. BLE 安全方案

### 5.1 产品识别（防止冒牌设备）

```
广播包结构:
┌──────────────────────────────────────────────────┐
│  AD Type: Complete List of 128-bit Service UUIDs │
│  0000ff00-0000-1000-8000-00805f9b34fb             │  ← 硬件层过滤依据
├──────────────────────────────────────────────────┤
│  AD Type: Complete Local Name                    │
│  "KeyGo-A3F29C"                                   │  ← MAC 后 3 字节动态生成
├──────────────────────────────────────────────────┤
│  AD Type: Manufacturer Specific Data             │
│  ID: 0xFFFF  Data: "KG"                           │  ← 二次确认标识
└──────────────────────────────────────────────────┘

App 端扫描:
  uni.startBluetoothDevicesDiscovery({
    services: ['0000FF00-0000-1000-8000-00805F9B34FB']
  })
  → OS 级过滤：蓝牙芯片只上报广播了此 UUID 的设备
  → 即使有同前缀名字的设备也不会出现
```

### 5.2 唯一设备名

```
所有设备刷同一份固件，开机时动态生成:

  uint8_t mac[6];
  esp_efuse_mac_get_default(mac);
  snprintf(name, 32, "KeyGo-%02X%02X%02X", mac[3], mac[4], mac[5]);
  → "KeyGo-A3F29C"

碰撞概率: 1677万种可能，小批量生产基本不碰撞
外壳二维码: 工厂测试夹具读 MAC → 生成二维码 → 打印贴纸
```

### 5.3 MAC 白名单绑定

```
首次使用（未绑定）:
  ┌────────┐                    ┌────────┐
  │  手机   │  ──扫描+连接──→    │ ESP32  │
  │        │  ←──FF02 Notify── │        │  {"bound":0, "authorized":1}
  │        │  ──FF03 BIND──→   │        │  保存手机 MAC 到 NVS
  │        │  ←──FF02 Notify── │        │  {"bound":1}
  └────────┘                    └────────┘
  → 仅此手机可控制，其他手机连接会被自动断开

已绑定后新手机连接:
  ┌────────┐                    ┌────────┐
  │ 新手机  │  ──连接──→        │ ESP32  │  MAC 不在白名单
  │        │  ←──断开──         │        │  "unauthorized device"
  └────────┘                    └────────┘

添加新手机（物理按键配对模式）:
  1. 按 PIN 9 的物理按键（< 2s）→ 进入配对模式 30 秒
  2. LED 快速闪烁（200ms 周期）
  3. 此时任何手机可连接并发送 BIND → 新的 MAC 写入白名单（覆盖旧的）
  4. 超时或成功绑定后退出配对模式

恢复出厂:
  1. 长按 PIN 9（> 5s）
  2. LED 三闪 → 清除 NVS 所有数据 → 重启
  3. 重新变为「未绑定」状态，首次连接者自动成为主人
```

### 5.4 安全层次对比

| 安全级别 | 方案 | 状态 |
|---------|------|------|
| Level 0 | 无任何安全（v2.1 之前） | 🔴 已淘汰 |
| Level 1 | MAC 白名单（v2.2） | 🟢 当前 |
| Level 2 | + BLE 配对加密 (LE Secure Connections) | 📋 规划 v2.3 |
| Level 3 | + 应用层 Token 鉴权 | 📋 规划 v2.4 |
| Level 4 | + 钥匙分享 + 撤销 | 📋 远期 |

> **v2.2 定位**: 民用级安全，阻止路人随意连接和控制。对专业攻击不构成绝对防护但已足够实用。

## 6. App 应用设计

### 6.1 技术栈

- **框架**: uni-app (Vue 3 + Pinia)
- **目标平台**: iOS / Android / 微信小程序
- **BLE 底层**: uni-app BLE API

### 6.2 页面结构

```
├─ index.vue     首页: 连接状态 + 设备列表 + 快捷操作 + 绑定入口
├─ config.vue    配置: RSSI 阈值/次数滑块 + 下发到设备
└─ ...
```

### 6.3 状态字段 (Store)

```js
state: {
  connected: bool,          // 是否已连接
  isBound: bool,            // 设备是否已绑定
  isAuthorized: bool,       // 当前连接是否通过白名单
  pairingMode: bool,        // 设备是否在配对模式
  deviceId: string,         // 当前连接的设备 ID
  deviceName: string,       // 设备唯一名 "KeyGo-A3F29C"
  deviceState: string,      // LOCKED / UNLOCKED / ACTION
  rssi: number,             // 原始 RSSI
  filteredRssi: number,     // Kalman 滤波后 RSSI
  ...
}
```

### 6.4 绑定流程 UX

```
首次使用:
  [App 打开] → 自动扫描 → 发现 KeyGo-A3F29C → 点击连接
  → 连接成功 → 显示 "未绑定" 标签 → 提示 "🔐 设备未绑定"
  → 点击 "锁定此设备" → 发送 BIND → 确认 → 显示 "已绑定"
  → 断开连接后，仅此手机可连接

已绑定后再次使用:
  [App 打开] → 自动重连已保存设备 → 直接进入控制面板
  → "已绑定" 标签显示

需要换手机:
  方案 A: 旧手机 App 里点 "解除绑定" → 新手机连接+绑定
  方案 B: 长按 PIN 9 物理按键 5s → 恢复出厂 → 新手机连接+绑定
```

## 7. 硬件 Pin Map

| Pin | 方向 | 用途 | 说明 |
|-----|------|------|------|
| 2 | OUTPUT | UNLOCK 继电器 | NMOS 高电平触发 |
| 3 | OUTPUT | LOCK 继电器 | NMOS 高电平触发 |
| 4 | OUTPUT | TRUNK 继电器 | NMOS 高电平触发 |
| 5 | OUTPUT | 钥匙供电 PMOS | 低电平导通 |
| 8 | OUTPUT | LED 状态指示 | 常亮/慢闪/快闪 |
| 9 | INPUT | 配对/复位按键 | INPUT_PULLUP, 低电平有效 |

## 8. 通信协议

### 8.1 GATT 服务

```
Service UUID:  0000FF00-0000-1000-8000-00805F9B34FB

Characteristic: 0000FF01  Write  (配置/RSSI注入)
Characteristic: 0000FF02  Read + Notify  (状态上报)
Characteristic: 0000FF03  Write  (手动命令)
```

### 8.2 FF01 配置写入

```
格式: key1=value1 key2=value2 ...

示例:
  unlock=-45 lock=-65 hyst=5 spike=25 uc=3 lc=5 interval=500 dlock=5000 q=4 r=16
  rssi=-40                    (手动注入 RSSI 用于测试)
  bound_mac=AA:BB:CC:DD:EE:FF (调试用直接设置绑定 MAC)
```

### 8.3 FF02 状态 JSON (1s Notify)

```json
{
  "connected": 1,
  "authorized": 1,        // ★ v2.2: 连接是否通过白名单
  "bound": 1,             // ★ v2.2: 设备是否已绑定
  "state": "LOCKED",
  "rssi": -52,
  "filtered": -50,
  "unlock": -45,
  "lock": -65,
  "hyst": 5,
  "uc": 3,
  "lc": 5,
  "manualCooldown": 0,
  "deviceName": "KeyGo-A3F29C",   // ★ v2.2
  "pairingMode": 0               // ★ v2.2
}
```

### 8.4 FF03 命令

| 命令 | 需鉴权 | 功能 |
|------|--------|------|
| `UNLOCK` | ✅ | 手动解锁 |
| `LOCK` | ✅ | 手动锁车 |
| `TRUNK` | ✅ | 开启后备箱 |
| `STATUS` | ✅ | 触发状态上报 |
| `BIND` | ❌ (开放) | 绑定当前连接的设备 MAC |
| `UNBIND` | ✅ | 清除绑定信息 |
