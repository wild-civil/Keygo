```
│              BLE广播 / 连接 / RSSI上报                    │

│  │CH582M   │          │ TMUX1112     │         │ (3个按键+   │ │
│  │         │          │              │         │  1路供电)   │ │
│  └────┬────┘          └──────────────┘         └────────────┘ │ │
│       │                                                     │
│  ┌────┴────┐                                               │
│  │电源管理  │ ← 18650锂电池 ← 太阳能板                      │
│  │TP4056+  │                                               │
│  │LDO 3.3V │                                               │
│  └─────────┘                                               │
└──────────────────────────────────────────────────────────┘
```

### 核心工作流程

```
手机靠近 → BLE连接建立 → RSSI持续采集 → 超过"解锁阈值"且持续N秒
  → 模拟开关闭合(解锁键) → 车门解锁

手机远离 → RSSI持续下降 → 低于"锁车阈值"且持续M秒
  → 模拟开关闭合(锁车键) → 车门锁定
```

## 2. MOS方案 vs 模拟开关方案 — 详细对比

### 2.1 原理差异

| 维度                | MOSFET方案                      | 模拟开关IC方案                   |
| ------------------- | ------------------------------- | -------------------------------- |
| **原理**            | MOS导通→漏源极连通→等效按键闭合 | IC内部集成双MOS+电荷泵，双向导通 |
| **开关方向**        | **单向**（体二极管决定方向）    | **双向**（无体二极管方向限制）   |
| **关断隔离**        | 体二极管可能反向漏电            | 双向完全隔离                     |
| **4路按钮所需元件** | 8颗MOS（每路2颗背靠背）         | **1颗IC**（4通道集成）           |
| **PCB面积**         | 大（8颗SOT-23）                 | **小**（1颗TSSOP-16）            |
| **BOM成本**         | MOS约0.3元×8=2.4元              | TMUX1112约3-5元/颗               |
| **设计复杂度**      | 高（需处理体二极管、Vgs偏置）   | **低**（VDD供电即可）            |

### 2.2 为什么MOS方案有"体二极管陷阱"

```
        N-MOS体二极管方向：
        
  漏极(D) ──→|── 源极(S)
              体二极管（S→D方向导通）
              
问题场景：钥匙按钮两端有直流偏压
  - 按钮端A：约2.5V（经过钥匙内部上拉）
  - 按钮端B：GND
  
  MOS关断时：
  - 如果D接端A，S接端B → 体二极管方向A→B，与偏压同向！
  - 偏压可能通过体二极管导通 → 按钮被意外触发！
  
  解决：背靠背双MOS（两管源极相连，体二极管方向相反）
  → 元件数量翻倍，布局更复杂
```

### 2.3 推荐方案：TMUX1112 模拟开关

**选定理由：**

1. **4通道SPST，一颗IC对应4路按钮**（解锁、锁车、后备箱、供电控制）
2. **Ron仅2Ω**，钥匙按钮电流μA级，压降可忽略
3. **漏电流仅3pA**，关断时完全隔离，绝无误触发风险
4. **双向信号路径**，不用关心按钮两端极性
5. **1.08-5.5V宽电压**，直接3.3V供电与MCU电平兼容
6. **先断后合（Break-before-make）**，防止按钮短路

| 参数         | TMUX1112     | 说明               |
| ------------ | ------------ | ------------------ |
| 配置         | 1:1 SPST × 4 | 4个独立常开开关    |
| 控制逻辑     | 高电平导通   | IN=1时开关闭合     |
| Ron(典型)    | 2Ω           | 远小于按键接触电阻 |
| 漏电流(最大) | 2nA          | 等效开路           |
| 供电范围     | 1.08-5.5V    | 3.3V直接供电       |
| 供电电流     | 9nA          | 几乎不耗电         |
| 封装         | TSSOP-16     | 5mm×6.4mm          |

### 3.3 最终推荐：CH582M

**推荐理由：**

1. **成本最优**：3.8元/颗，是nRF52810的1/3，ESP32-C3的2/3
2. **功耗极低**：0.5μA深睡，BLE连接保持仅10μA，18650电池可运行数月
3. **BLE5.3够用**：支持Coded PHY（长距离模式），RSSI精度与nRF持平
4. **集成USB**：免外部下载器，CH582M自带USB 2.0，一根线即可烧录+调试
5. **国产供应链**：沁恒WCH南京产线，无断供风险
6. **资源充足**：256KB Flash / 32KB RAM，本项目代码量<50KB，绰绰有余

**ESP32-C3作为备选的场景：**

- 如果你想先用Arduino快速验证BLE连接和RSSI逻辑，ESP32-C3开发最快
- 验证通过后，固件逻辑迁移到CH582M工作量不大（都是BLE GATT操作）

## 4. 五菱缤果车钥匙改装要点

### 4.1 钥匙基本信息

- **电池**：CR2032（3V纽扣电池）
- **按键**：3个（解锁、锁车、后备箱），导电橡胶触点式
- **射频**：433MHz（中国区标准），滚动码加密
- **壳体**：超声波焊接，需沿中缝撬开

### 4.2 按键触点的电气特征（推测+通用规律）

五菱缤果钥匙内部结构属于典型低遥控钥匙：

```
钥匙PCB内部按键等效电路：

       VCC(3V)
        │
       [上拉电阻 ~10kΩ]
        │
   ┌────┤──── 按键触点A → 连接TMUX1112的S1引脚
   │    │
  [橡胶] │
   │    │
   └────┤──── 按键触点B → GND
        │
       GND
       
按下时：触点A与触点B导通（电阻~100Ω-1kΩ）
       → 触点A被拉到GND → 钥匙MCU检测到低电平 → 发射RF信号
```

**关键操作**：

1. 拆开钥匙，找到3个按键对应的4个焊盘（每个按键2个焊盘）
2. 用万用表确认：哪个焊盘是GND，哪个是信号端
3. 信号端焊飞线连接到TMUX1112的Sx引脚
4. GND端焊飞线连接到TMUX1112的Dx引脚
5. TMUX1112导通时，Sx-Dx连通，等效于按键按下

> ⚠️ **重要**：信号端和GND端接反没关系！TMUX1112是双向开关，任意方向都能导通。

### 4.3 供电控制（MOS4 → 第4路模拟开关）

"断开原车钥匙电池，由控制板控制供电"：

```
原车钥匙电池正极 ──┬── 切断 ──┐
                   │           │
              原来直连3V    改为：TMUX1112第4通道
                               │
                   控制板3.3V ──S4──D4──→ 钥匙VCC
                               │
                         IN4=1时导通，钥匙上电
                         IN4=0时断开，钥匙断电
```

**注意**：钥匙MCU上电后需要启动时间（约50-200ms），不能立即触发按键。软件流程：

```
解锁流程：
1. RSSI超过阈值 → IN4=1（钥匙上电）
2. 延时200ms（等待钥匙MCU初始化）
3. IN1=1（模拟解锁键按下）
4. 延时150ms（模拟按压时长）
5. IN1=0（释放按键）
6. 延时500ms（等待RF发射完成）
7. IN4=0（钥匙断电，省电）
```

## 5. 电源方案设计

### 5.1 电源树

```
太阳能板(5V/1W) ──→ TP4056 ──→ 18650锂电池(3.7V) ──→ ME6211(LDO) ──→ 3.3V主电源
                      ↑充电                                │
                    充电保护                              ├── CH582M VDD
                    过充4.2V                              ├── TMUX1112 VDD
                    过放2.8V                              └── 太阳能板采样（ADC）
```

### 5.2 关键参数估算

| 负载           | 工作模式  | 电流   | 占空比 | 日均功耗      |
| -------------- | --------- | ------ | ------ | ------------- |
| CH582M BLE扫描 | 广播+扫描 | ~1.5mA | 100%   | 36mAh/天      |
| CH582M BLE连接 | 连接保持  | ~10μA  | ~80%   | 0.19mAh/天    |
| TMUX1112       | 常通      | 9nA    | 100%   | 可忽略        |
| 钥匙上电       | 按键发射  | ~15mA  | <0.1%  | 可忽略        |
| **系统总计**   |           |        |        | **~36mAh/天** |

- **18650电池容量**：2500mAh（典型值）
- **纯电池续航**：2500 / 36 ≈ **70天**（持续BLE扫描模式）
- **BLE连接后降功耗**：连接建立后切换到连接保持，功耗降至 ~0.2mAh/天 → 理论续航 **3年+**
- **太阳能补电**：5V/1W板，假设日均有效光照2小时 → 补电 ~400mAh/天 → **完全覆盖日常功耗**

> 💡 **优化策略**：设备安装在车内，手机不在附近时BLE处于广播等待模式（~1.5mA）。手机连接后切换到连接保持模式（~10μA），配合太阳能几乎永不断电。

## 6. RSSI测距算法设计

### 6.1 RSSI与距离的关系

距离估算公式（对数距离路径损耗模型）：

```
d = 10^((A - RSSI) / (10 × n))
```

- d：距离（米）
- A：1米处RSSI参考值（需实测校准，典型值-40~-60dBm）
- RSSI：实测信号强度
- n：环境衰减因子（开放空间2.0，车内3.0-4.0）

### 6.2 阈值与滞后策略

```
RSSI (dBm)
  0 ─────────────────────────────────
 -40 ────────┐                       
 -50         │← "进入区"             
 -60 ────────┤   RSSI > -60 持续3秒  ──→ 触发解锁
 -65         │                       
 -70 ────────┤   RSSI < -70 持续5秒  ──→ 触发锁车
 -75         │← "退出区"             
 -80 ────────┘                       
-100 ─────────────────────────────────
             │← 滞后区(-60 ~ -70)    
             │  在此区间不触发任何动作  
             │  防止反复解锁/锁车      
```

**核心算法要点：**

1. **双阈值滞后**：进入阈值(-60dBm) ≠ 退出阈值(-70dBm)，中间区域不动作
2. **滑动窗口**：连续N次RSSI采样都超阈值才触发（N=5~10，采样间隔200ms）
3. **超时保护**：解锁后30秒无车门动作 → 自动锁车
4. **手动模式**：APP按钮直接触发解锁/锁车，不依赖RSSI

## 9. 风险与注意事项

| 风险         | 影响       | 缓解措施                 |
| ---------- | -------- | -------------------- |
| RSSI波动大    | 误触发解锁/锁车 | 双阈值+滑动窗口+超时保护        |
| 钥匙MCU上电时序  | 按键无效     | 上电后延时200ms再触发按键      |
| 钥匙PCB焊接不可逆 | 改装失败无法恢复 | 先用备用钥匙测试；焊接前拍照记录     |
| 车内高温       | 电池安全     | 18650加温度保护；太阳能板选耐温型号 |
| BLE断连      | 无法解锁     | 保留原钥匙机械按键可用；APP手动模式  |
| 太阳能板积灰     | 充电不足     | 18650 纯电续航~70天，足够缓冲   |
