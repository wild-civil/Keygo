## 一、项目背景与目标

### 1.1 我要做什么

为五菱缤果汽车制作基于BLE（低功耗蓝牙）的无感舒适进入系统：

- **靠近车辆**（约3米）→ 自动解锁
- **远离车辆**（约5米）→ 自动锁车
- 用户只需要携带手机，无需操作任何按键

### 1.2 原车钥匙分析

| 项目     | 详情                                           |
| -------- | ---------------------------------------------- |
| 钥匙类型 | 433MHz无线射频钥匙                             |
| 电池     | CR2032纽扣电池（3V）                           |
| 按键     | 3个（解锁/锁车/后备箱）                        |
| 触发方式 | 低电平触发（按键一端=GND，另一端=MCU上拉输入） |
| 加密方式 | 滚动码加密                                     |

### 1.3 核心思路

> **保留原车钥匙不动**，用MOS管"并联"在钥匙按键两端，模拟人手按键动作，通过BLE信号判断距离后自动触发。

## 三、方案演变历程

### 3.1 方案A（已验证成功）：主机扫描模式

| 项目                                                        | 内容                                              |
| ----------------------------------------------------------- | ------------------------------------------------- |
| 角色                                                        | ESP32做**主机**，主动扫描周围BLE设备              |
| 原理                                                        | 扫描手机/蓝牙耳机等设备的广播包，提取RSSI判断距离 |
| 验证结果                                                    | ✅ **已验证成功**                                  |
| 用蓝牙耳机验证：RSSI=-31dBm（近距离），-80~-99dBm（远距离） |                                                   |
| 解锁/锁车/超时保护全部正常工作                              |                                                   |

**方案A的问题：**

- ❌ **手机BLE信号不可见**：现代手机（Android/iOS）出于隐私保护，不会主动持续广播BLE信号，ESP32扫描不到手机
- ❌ 需要额外App（如nRF Connect）让手机强制广播，用户体验差
- ❌ 手机BLE MAC地址会随机化变化，即使扫到也无法稳定绑定

### 3.2 方案B（当前确认）：从机广播 + 设备端RSSI测距 ✅

| 项目   | 内容                                                                                                                                |
| ---- | --------------------------------------------------------------------------------------------------------------------------------- |
| 角色   | ESP32C3做**从机**，持续广播自己的 BLE 信号                                                                                                     |
| 原理   | 手机 App 连接 ESP32C3 后，**设备端测量 RSSI**判断距离，自主执行解锁/锁车                                                                                  |
| 参考实现 | https://github.com/MarkDing/IoT-Developer-Boot-Camp/wiki/Bluetooth-Smart-Key-and-Demo-cn#33-demo 和 PAN1080 BLE Vehicle Key（磐启微电子） |

**为什么这个方案更好：**

- ✅ **ESP32C3的 MAC 地址固定不变**（Public Address），手机可以稳定识别
- ✅ 设备端自主测距、自主执行，不依赖手机App持续运行
- ✅ App设置好后可以切后台，不需要一直打开
- ✅ 符合真实车钥匙产品的工作模式

### 4.2 RSSI测距原理

**RSSI（Received Signal Strength Indicator）** — 接收信号强度指示

```
距离越近 → RSSI值越大（接近0）→ 信号越强
距离越远 → RSSI值越小（接近-100）→ 信号越弱
```

| 距离估算        | 典型RSSI范围   | 场景                   |
| --------------- | -------------- | ---------------------- |
| 贴身（<0.5m）   | -10 ~ -30 dBm  | 手机放在口袋里贴着设备 |
| 近距离（~1-2m） | -30 ~ -50 dBm  | **解锁区域**           |
| 中距离（~3-5m） | -50 ~ -70 dBm  | **锁车区域**           |
| 远距离（>5m）   | -70 ~ -100 dBm | 已远离车辆             |

> **注意**：RSSI受人体遮挡、金属反射、天线方向等影响波动很大，必须用**迟滞阈值 + 连续确认次数**来防抖。

### 4.3 迟滞阈值设计（防抖）

```
RSSI值
  0 ─┬────────────────────────────────────
     │
-30 ─┤ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ← 解锁阈值（rssi_unlock）
     │            ↑ 连续5次≥此值→解锁      │
-50 ─┤ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ← 锁车阈值（rssi_lock）
     │            ↑ 连续10次≤此值→锁车     │
-70 ─┤                                     │
     │                                     │
-100└─────────────────────────────────────┘

迟滞区间（-30 ~ -50之间的"灰色地带"）防止：
- 站在车边RSSI波动 → 反复解锁/锁车
- 信号偶尔跳动 → 误触发
```

### 4.4 参考实现对标（PAN1080 BLE Vehicle Key）

磐启微电子的PAN1080方案是成熟的BLE车钥匙参考设计：

| PAN1080参数                       | 我们的对应         |
| --------------------------------- | ------------------ |
| `vcmd rssi_open_thr`（开锁阈值）  | `rssi_unlock` 变量 |
| `vcmd rssi_close_thr`（关锁阈值） | `rssi_lock` 变量   |
| 设备广播→手机连接→设备测RSSI      | 完全一致           |
| 设备端自主判断、自主执行          | 完全一致           |

**关键确认：PAN1080方案中，RSSI是在设备端（从机）测量的，不是手机端！**

```
Service UUID: 0xFF00 (自定义服务)
```

## 八、已知问题与应对

### 8.1 RSSI波动问题

- **现象**：RSSI值跳动大（同一位置可能跳±15dBm）
- **对策**：迟滞阈值 + 连续N次确认 + 滑动平均滤波

### 8.2 蓝牙连接断开

- **现象**：手机锁屏后BLE可能断连
- **对策**：ESP32检测到断连后自动重新广播，手机App自动重连（BLE协议支持自动重连）

### 8.3 多设备干扰

- **现象**：周围BLE设备很多（扫描到100+个）
- **对策**：从机模式下只有连接到本设备的手机才会触发，天然过滤其他设备

### 8.4 ESP32 Arduino Core 3.x兼容

- **已解决**：BLE API使用`getScan()`单例模式
- **Flash Mode**：必须使用DIO模式

## 十一、当前无App验证步骤

当前阶段先不做正式App，使用 `code/BLE_Key_Go_v2/BLE_Key_Go_v2.ino` + nRF Connect 验证。

### 11.1 烧录前检查

1. Arduino IDE选择 `ESP32C3 Dev Module`
2. Flash Mode选择 `DIO`
3. 先不要接原车钥匙，只接LED或万用表验证GPIO动作
4. 串口波特率 `115200`

### 11.2 nRF Connect连接

1. 打开手机蓝牙
2. 打开 nRF Connect
3. 扫描并连接 `BLE-Key-Go-v2`
4. 找到自定义服务 `FF00`

| UUID | 方向 | 用途 |
| ---- | ---- | ---- |
| `FF01` | Write | 写入配置 |
| `FF02` | Read/Notify | 读取状态、RSSI、阈值 |
| `FF03` | Write | 手动命令 |

### 11.3 可写入内容

配置写入 `FF01`：

```text
unlock=-45 lock=-65 uc=5 lc=10 interval=500 dlock=5000
```

如果当前 Arduino/ESP32 环境不支持设备端原生读取连接 RSSI，固件会进入手动 RSSI 注入模式。此时可以写入：

```text
rssi=-40
rssi=-80
```

用来先验证迟滞阈值、连续确认、MOS按键时序是否正常。

手动命令写入 `FF03`：

```text
UNLOCK
LOCK
TRUNK
STATUS
```

### 11.4 验证目标

1. 手机连接后，串口能持续输出 RSSI
2. 手机贴近设备时 RSSI 明显变大
3. 手机远离设备时 RSSI 明显变小
4. 调整 `unlock` / `lock` 阈值后，能触发解锁/锁车序列
5. 记录车内、车外、1m、3m、5m 的 RSSI 数据，再决定正式阈值

如果串口提示 `Native ESP GAP RSSI: unavailable`，说明当前板卡包没有暴露底层RSSI读取API。此时先用 `rssi=-40` / `rssi=-80` 验证控制逻辑，后续再切换到支持连接RSSI的BLE库或板卡包。

### 11.5 重要边界

这个阶段只能验证“连接后设备端RSSI测距是否可用”。它还不能证明最终能做到后台无感，因为后台自动连接需要后续原生App验证。
