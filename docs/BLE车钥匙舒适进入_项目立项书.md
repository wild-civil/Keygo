
# 🚗 BLE车钥匙舒适进入系统 — 项目立项书

> **项目代号**：BLE-Key-Go  
> **立项日期**：2026-06-23  
> **目标车型**：五菱缤果  
> **项目性质**：个人DIY — 低成本复刻商用"蓝牙舒适进入"功能

---

## 一、项目目标

自制一套基于**BLE RSSI测距**的无感车钥匙舒适进入系统，实现：

| 功能                    | 描述                                   |
| ----------------------- | -------------------------------------- |
| 🔓 **靠近自动解锁**      | 手机靠近车辆1-3m，自动触发原车钥匙解锁 |
| 🔒 **远离自动锁车**      | 手机远离车辆5m+，自动触发原车钥匙锁车  |
| 📦 **后备箱控制**        | 预留后备箱按钮通道，可由App触发        |
| ☀️ **太阳能+锂电池供电** | 18650+太阳能板，安装即用无需频繁充电   |
| 📱 **后期App扩展**       | 二期微信小程序配对/手动控制/阈值调节   |

**核心价值**：商用产品200-500元，本项目 BOM 成本 **~33元** ，相对自主可控。

## 二、系统架构

```
┌───────────────────────────────────────────────────────────┐
│  手机（BLE Peripheral/广播模式）                           │
│  • 蓝牙常开，无需后台App                                   │
│  • 一期：仅做BLE广播源（设备端扫描RSSI）                    │
│  • 二期：微信小程序前台BLE连接（配对/手动解锁/阈值调节）    │
└──────────────────────┬────────────────────────────────────┘
                       │ 2.4GHz BLE
                       ▼
┌───────────────────────────────────────────────────────────┐
│  控制板（安装于车内，贴仪表台/中控台）                     │
│                                                           │
│  ┌──────────┐  GPIO  ┌──────────────┐  飞线  ┌──────────┐ │
│  │BLE主控    │──────→│ 4通道模拟开关  │─────→│原车钥匙PCB│ │
│  │CH582M    │       │ TMUX1112     │      │(3键+供电) │ │
│  │(Central) │       │              │      └──────────┘ │ │
│  └─────┬────┘       └──────────────┘                    │ │
│        │ RSSI采集 + 双阈值判断 + 状态机                   │ │
│  ┌─────┴─────┐                                          │ │
│  │电源管理    │ ← 18650 ← 太阳能5V/1W                   │ │
│  │TP4056+LDO │                                          │ │
│  └───────────┘                                          │ │
└───────────────────────────────────────────────────────────┘
```

### 核心设计决策：**设备做Central**

| 维度     | 设备做Central（✅已选定）           | 手机做Central         |
| -------- | ---------------------------------- | --------------------- |
| RSSI采集 | 设备端完成，手机零参与             | 需手机后台App持续运行 |
| 无感程度 | ✅ 真正无感（手机蓝牙开着即可）     | ❌ 手机需后台进程      |
| 首次配对 | 串口/AT指令（一期）→小程序（二期） | App扫描连接           |
| 功耗     | 设备端功耗可控（18650+太阳能覆盖） | 手机端持续扫描耗电    |
| 安全性   | 设备端白名单，只认绑定手机         | 手机端需安全存储密钥  |

## 三、技术方案概要

### 3.1 车钥匙改装方案

```
原车钥匙（五菱缤果 CR2032 3V 433MHz滚动码）
  ├── 按键类型：低电平触发（一端GND，一端MCU上拉检测）
  ├── 改装方式：断开原电池，4路飞线引出
  │     ├── CH1 → 解锁键信号端
  │     ├── CH2 → 锁车键信号端
  │     ├── CH3 → 后备箱键信号端
  │     └── CH4 → 钥匙VCC供电控制
  └── 按键时序：上电200ms → 按键150ms → 等RF发射500ms → 断电
```

### 3.2 开关方案

| 方案                                     | 元件数 | 成本  | 推荐场景                                   |
| ---------------------------------------- | ------ | ----- | ------------------------------------------ |
| **MOS方案**（3×N-MOS按键 + 1×P-MOS供电） | 4颗    | ~¥1.2 | ✅ 面包板验证阶段（分立元件好接线）         |
| **TMUX1112模拟开关**                     | 1颗IC  | ~¥4.5 | ✅ PCB正式版（集成度高、双向隔离、3pA漏电） |

> **面包板验证用 MOS，PCB 正式版可选用 TMUX1112**——后期分析成本和可靠性的平衡。

### 3.3 主控选型

| 用途               | 芯片     | 理由                                      |
| ------------------ | -------- | ----------------------------------------- |
| **面包板快速验证** | ESP32-C3 | 手上有开发板，Arduino生态最快出原型       |
| **正式产品**       | CH582M   | 3.8元/0.5μA深睡/BLE5.3/自带USB/国产供应链 |

### 3.4 电源方案

```
太阳能5V/1W → TP4056(充电保护) → 18650 2500mAh → ME6211 LDO 3.3V
```

| 指标                      | 数值                |
| ------------------------- | ------------------- |
| 纯电池续航（BLE扫描模式） | ~70天               |
| 纯电池续航（BLE连接模式） | ~3年+               |
| 太阳能日均补电            | ~400mAh（完全覆盖） |

### 3.5 RSSI算法

- **双阈值滞后**：解锁-60dBm / 锁车-70dBm，中间10dBm死区防抖
- **滑动窗口**：连续5-10次采样超阈值才触发
- **超时保护**：解锁30秒无车门动作→自动锁车
- **后期可升级**：可加入IMU辅助判断（加速度变化→车辆移动→防误锁）

## 四、三期开发规划

### 一期：硬件闭环验证（目标：2-3周）

> **核心目标**：不写App，纯硬件验证"BLE RSSI → 自动解锁/锁车"链路能跑通

| 任务                     | 产出                     | 预计工时 |
| ------------------------ | ------------------------ | -------- |
| ESP32-C3面包板接线       | 面包板+MOS+飞线→原车钥匙 | 2h       |
| Arduino BLE扫描+RSSI采集 | 串口输出RSSI实时数据     | 4h       |
| 双阈值状态机代码         | 自动解锁/锁车逻辑        | 4h       |
| 钥匙拆解+触点确认        | 万用表确认GND/信号端     | 1h       |
| 飞线焊接+端到端测试      | 靠近解锁/远离锁车成功    | 3h       |
| **小计**                 |                          | **~14h** |

**交付物**：

- ESP32-C3 Arduino代码（BLE扫描+RSSI+状态机）
- 面包板接线照片+钥匙改装记录
- RSSI阈值调参数据（车内/车外各距离实测）

**一期不做**：

- ❌ 不做手机App
- ❌ 不做NFC
- ❌ 不做PCB

**首次配对方式**：串口AT指令绑定手机蓝牙MAC地址（写入白名单）

---

### 二期：微信小程序 + CH582M PCB（目标：4-6周）

> **核心目标**：从面包板迁移到正式PCB，微信小程序替代串口配对

| 任务                | 产出                                       |

| 整机集成测试        | 安装到车内实测                             |
| 3D打印外壳          | 适配车内安装位                             |

**交付物**：

- CH582M PCB Gerber文件

- 整机安装照片+实测数据

## 六、风险与应对

| 风险                  | 影响          | 概率 | 应对                               |
| --------------------- | ------------- | ---- | ---------------------------------- |
| RSSI波动大导致误触发  | 频繁解锁/锁车 | 中   | 双阈值+滑动窗口+超时保护，实测调参 |
| 钥匙MCU上电时序不明确 | 按键无效      | 低   | 上电后延时200ms，可调参            |
| 钥匙焊接不可逆        | 改装失败      | 中   | 先用备用钥匙测试；焊前拍照         |
| 车内高温影响电池      | 安全隐患      | 低   | 18650带温度保护；仪表台温度可控    |
| BLE意外断连           | 无法解锁      | 低   | 保留机械钥匙可用；App手动模式兜底  |
| 太阳能积灰充电不足    | 长期断电      | 低   | 纯电池续航70天缓冲；定期擦拭       |
