# CH582M 小白入门指南

> 从零开始，三步搞定 CH582M 开发

---

## 你需要准备的东西

| 物品 | 说明 | 大概价格 |
|------|------|---------|
| CH582M 开发板 | 淘宝搜"CH582M 开发板"，选带 WCH-Link 调试器的 | ¥20-40 |

## 第一步：安装开发环境（MounRiver Studio）

### 1. 下载 MounRiver Studio

官网：http://www.mounriver.com/download

下载 **MounRiver Studio Ⅱ (MRS 2)** 版本（VSCode 风格的那个）。

### 2. 安装

无脑下一步就行，安装路径不要有中文和空格。

### 3. 打开软件

第一次打开可能会慢一点，耐心等。

---

## 第二步：让 LED 闪起来（验证环境）

这一步的目标：**确认你能编译、烧录、运行程序**。

### 2.1 打开工程

1. 打开 MounRiver Studio
2. 菜单：`File` → `Open` → `Projects from File System`
3. 选择这个工程文件夹：`CH582M_BLE_Slave`
4. 点击 `Finish`

### 2.2 编译

点击工具栏上的 **锤子图标**（Build），或者按 `F7`。

如果底部控制台出现 `Build Finished. 0 errors, 0 warnings`，就说明编译成功了！

### 2.3 连接开发板

1. 用 USB 线把 WCH-Link 连到电脑
2. WCH-Link 的排线接到开发板上（注意方向，红线对应 VCC）

### 2.4 烧录程序

点击工具栏上的 **下载图标**（Flash 图标），或者按 `F8`。

如果底部显示 `Download Successfully`，就说明烧录成功了！

### 2.5 看效果

开发板上的 LED 应该开始**每秒闪一次**（亮 500ms，灭 500ms）。

> ⚠️ **注意**：不同开发板的 LED 引脚可能不一样！
> - 如果你的开发板 LED 不闪，可能是引脚不对。
> - 看看开发板背面的丝印，LED 接在哪个引脚上。
> - 然后修改 `src/Main.c` 里的 `LED_PIN` 和 `LED_PORT`。

## 第三步：跑 BLE 广播（用官方例程）

CH582M 的 BLE 协议栈是沁恒官方提供的，必须从官方 SDK 里拿。

### 3.1 下载官方 SDK

去沁恒官网下载 CH58x EVT 包：
- 搜索 "CH582 EVT" 或 "CH583 EVT"
- 下载地址：https://www.wch.cn/downloads/CH583EVT_ZIP.html

下载后解压，你会看到这样的目录结构：
```
CH583EVT/
├── EVT/
│   ├── EXAM/          ← 各种例程
│   │   └── BLE/       ← BLE 相关例程
│   │       ├── Peripheral/   ← 【我们要用这个】蓝牙从机例程
│   │       ├── Central/      ← 蓝牙主机例程
│   │       ├── Broadcaster/  ← 广播者例程
│   │       └── ...
│   └── PUB/           ← 公共文件（BLE 库、头文件等）
└── ...
```

### 3.2 打开官方 Peripheral 例程

1. 在 MounRiver Studio 里
2. `File` → `Open` → `Projects from File System`
3. 选择 `CH583EVT/EVT/EXAM/BLE/Peripheral` 文件夹
4. 点击 `Finish`

### 3.3 编译烧录

和第二步一样：
- 按 `F7` 编译
- 按 `F8` 烧录

### 3.4 用手机验证

1. 在手机上安装 **BLE 调试助手**（随便搜一个，比如 "BLE调试宝"、"nRF Connect"）
2. 打开 APP，开始扫描
3. 你应该能搜到一个叫 **"Simple Peripheral"** 的设备
4. 点进去，能看到各种服务和特征值

🎉 **恭喜你！BLE 广播跑通了！**

## 第四步：改成我们自己的设备名

既然跑通了，我们来改点东西，体验一下修改代码的感觉。

### 4.1 找到修改广播名称的地方

在 `Peripheral` 例程里，找到 `APP` 文件夹下的 `peripheral.c` 文件。

找到 `scanRspData` 这个数组（大概在文件开头位置）：

```c
static uint8_t scanRspData[] =
{
  // complete name
  0x12,   // length of this data
  GAP_ADTYPE_LOCAL_NAME_COMPLETE,
  'S','i','m','p','l','e',' ','P','e','r','i','p','h','e','r','a','l',
  
  // ... 后面还有其他数据
};
```

### 4.2 修改设备名

把 `'S','i','m','p','l','e',' ','P','e','r','i','p','h','e','r','a','l'` 改成你想要的名字，比如 `'B','L','E','-','K','e','y','-','G','o'`。

**注意长度也要改！** 原来的 `0x12`（十进制 18）是 "Simple Peripheral" 的长度（17个字符 + 1个Type字节 = 18）。如果你的新名字是 8 个字符，长度就要改成 `0x09`（8 + 1 = 9）。

举个例子，改成 "BLE-Key-Go"：
```c
static uint8_t scanRspData[] =
{
  // complete name
  0x0A,   // length = 1(类型) + 9(名字长度) = 10 = 0x0A
  GAP_ADTYPE_LOCAL_NAME_COMPLETE,
  'B','L','E','-','K','e','y','-','G','o',
  
  // ... 后面的保持不变
};
```

### 4.3 重新编译烧录

- `F7` 编译
- `F8` 烧录
- 手机重新扫描，看看名字是不是变了！

## 常见问题 FAQ

### Q: 编译报错怎么办？
A: 先看错误信息，最常见的原因：
- 工程路径有中文（比如放在桌面/下载里，用户名是中文）
- 没有选对芯片型号（工程里选的是 CH582M，但 SDK 是 CH583 的？其实通用，不用管）

### Q: 烧录失败怎么办？
A: 检查：
- WCH-Link 插好了吗？
- 排线方向对吗？（红线对 VCC）
- 开发板通电了吗？
- 驱动装了吗？（MounRiver Studio 自带驱动，一般不用手动装）

### Q: 手机搜不到设备怎么办？
A: 
1. 确认烧录成功了
2. 确认用的是 BLE 调试 APP（系统蓝牙设置里可能搜不到）
3. 试试重启手机蓝牙

### Q: ESP32 和 CH582M 开发感觉有啥不一样？
A: 
| 对比项 | ESP32 (Arduino) | CH582M (MRS) |
|--------|-----------------|--------------|
| 开发方式 | Arduino IDE，简单直观 | MounRiver Studio，专业 IDE |
| 代码结构 | setup() + loop() | main() + 各种初始化函数 |
| BLE 库 | 开源，ArduinoBLE 或 NimBLE | 闭源，官方 .a 库 |
| 调试方式 | 串口打印 | WCH-Link 调试 + 串口 |
| 资料 | 非常多 | 相对少一些 |
